Technical documentation

CrashlessLLM Architecture

CrashlessLLM is a managed safety boundary for local GGUF inference in .NET and Avalonia apps. The architecture is designed around one operational promise: native inference should fail predictably, stream safely, and release deterministically instead of freezing or killing the host UI process.

Failure domain

The problem is not calling llama.cpp. The problem is surviving it.

High-performance local inference libraries allocate aggressively outside the CLR. When a .NET desktop app embeds that runtime directly, the application crosses from managed safety into a world where the GC cannot see model weights, KV cache, sampler state, backend workspaces, GPU offload allocations, or allocator fragmentation.

Naive native wrappers discover memory failure too late. Once the process is inside native model/context allocation, the failure can be an OS OOM kill, a native exception, a null pointer path, memory-pressure stalls, or a frozen UI message pump.
ConcernNaive wrapperCrashlessLLM
Model admissionLoad first, hope allocation succeeds.Predict memory pressure before native allocation.
Native memory visibilityCLR cannot see weights, context, sampler, or backend workspaces.Native telemetry is surfaced as managed diagnostics.
Token streamingCallbacks push at model speed regardless of UI consumption.Bounded channel backpressure forces the worker to wait.
UTF-8 handlingToken fragments are often marshalled as independent strings.Bytes are copied and accumulated until valid UTF-8 can be emitted.
LifetimeRaw pointers or finalizer timing leak into app code.Opaque handles are wrapped in SafeHandle and disposed deterministically.
ConfigurationSampling, token limits, and prompt formats are scattered.SamplingOptions, MaxTokens, and ChatTemplate centralize behavior.
System map

The boundary is deliberately layered.

The architecture separates developer ergonomics from native survival mechanics. The public surface remains small while the internals enforce admission control, ABI safety, streaming pressure control, and deterministic cleanup.

Layer 2

Predictive Allocation Gating

CrashlessLLM reframes model load as an admission-control decision. The gate estimates whether the process can safely admit the requested model/context before calling into native model allocation.

Pre-load heuristic

K = C × 131,072
B = W + K
M = floor(B × r)
P = B + M

allow load if A is unavailable, otherwise require P <= A
SymbolMeaningImplementation
WGGUF model file size in bytesstat / _stat64
CRequested context size in tokensLlmLoadOptions.ContextSize, default 4096
DConservative KV density131,072 bytes/token (128 KiB)
KEstimated KV cache bytesC × D
rSafety margin fractiondefault 0.30
AAvailable physical memoryplatform telemetry
Why a conservative heuristic? The gate must run before llama.cpp has parsed model internals. It intentionally favors rejecting risky loads over entering native allocation and hoping the OS stays kind.

Layer 3

Accurate post-load KV introspection

Layer 5

Native sampler-chain construction

SamplingOptions translates managed intent into a native sampler chain. Omitted values use sentinel defaults so callers can tune only what they care about.

Managed optionNative sentinelBehavior
Temperature< 0Default 1.0; 0 selects greedy decoding.
TopK<= 0Default 40; 1 behaves like top-1 sampling.
TopP< 0Default 0.95.
MinP< 0Default 0.05.
RepeatPenalty< 0Default 1.0; non-1.0 values enable repeat penalty.
RepeatLastN<= 0Defaults to 64.
Seed0Native random-seed behavior.
Operational quirk: Temperature = 0 is not a tiny temperature. It intentionally switches to greedy decoding so deterministic workflows do not rely on sampler edge behavior.

Layers 7–8

GC-safe streaming and bounded backpressure

Native token callbacks are synchronous and ephemeral. The callback pointer is valid only during invocation, so CrashlessLLM copies native bytes immediately and never stores the native pointer.

1. Native workerGeneration runs away from the caller/UI thread.
2. Ephemeral callbackCallback receives a temporary const char* token piece.
3. Immediate byte copyManaged code copies bytes before the native buffer goes out of scope.
4. UTF-8 reconstructionIncomplete multi-byte sequences are buffered until valid.
5. Bounded channelAt most 50 tokens can sit ahead of the consumer.
6. Async UI consumptionAvalonia consumes IAsyncEnumerable<string> at its own pace.
The pressure valve is intentional. If the channel fills, the native worker waits. Backpressure lands on inference, not the UI message pump.
Layer 9

Deterministic ownership and concurrency model

CrashlessLLM treats native sessions as expensive, stateful resources with a strict release order. The managed wrapper exposes IDisposable; the native boundary releases sampler, context, model, worker state, and the session container.

llama_sampler*Native session containerFreed before context/model teardown.
llama_context*Native session containerFreed after sampler.
llama_model*Native session containerFreed after context.
Worker stateNative session containerCancelled before handle disposal; deferred cleanup avoids worker-thread join deadlocks.

Boundaries

Non-goals are part of the design.

Not a sandboxDo not load untrusted GGUF files. Parser vulnerabilities are below this boundary.
Not a serverCrashlessLLM is not Ollama, REST inference, or a cloud model host.
Not an exact pre-load proofThe admission gate is conservative. Accurate architecture reporting is available after load.
Not multi-stream per sessionCreate multiple sessions for parallel generation instead of multiplexing one native context.
Summary

A small API over a strict stability boundary.

CrashlessLLM turns local LLM embedding from an unmanaged allocation gamble into a predictable subsystem: memory admission, ABI safety, bounded streaming, deterministic native ownership, structured diagnostics, and reproducible packaging.
